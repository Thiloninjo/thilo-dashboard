Bekannte Bottlenecks & Risiken

Kritisch

Kein Auto-Restart lokal

Backend/Frontend starten via VBS hidden — wenn sie crashen, gibt es keine Fehlermeldung und keinen Neustart. Dashboard zeigt dann leere Daten oder lädt gar nicht.

Kritisch

Windows \r\n Parsing

Vault-Dateien auf Windows haben \r\n. Alle Regex-Parser brauchen .replace(/\r/g, "") — sonst matchen Checkbox-Patterns nicht. Heute gefixt in 3 Services.

Gelöst

Git-Sync jetzt via Webhook (~1-2s)

GitHub Webhook triggert sofortigen Pull auf Hetzner. 30s-Polling bleibt als Fallback. Einziger verbleibender Delay: lokale Änderungen müssen committed + gepusht werden.

Mittel

Kein Prod-Build Frontend

Das Dashboard-Frontend läuft nur als Vite Dev Server — kein Production Build, kein Server-Side Rendering, nicht auf Hetzner deployed. Nur lokal nutzbar.

Mittel

Uncommitted Changes

Viele lokale Änderungen sind nicht committed (api.ts Fallback-Logik, Cache-Updates, etc.). Hetzner hat eine ältere Codeversion als der lokale PC.

Info

Habits nur auf Hetzner

Die Habit-Daten (habits.json, habit-log.json) leben nur auf dem Hetzner Server. Kein Backup, kein Sync zum Vault. Bei Server-Verlust: Daten weg.
